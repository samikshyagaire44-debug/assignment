HTML Basics

Understanding the foundation of web development with HyperText Markup Language

What is HTML?

HTML (HyperText Markup Language) is the standard markup language used to create web pages and web applications. It provides the basic structure and content of websites by using a system of elements and tags that define different parts of a document, such as headings, paragraphs, links, images, and other content.

HTML is not a programming language but rather a markup language that uses tags to annotate text and define how content should be structured and displayed in web browsers. These tags tell the browser how to interpret and present the content to users, creating the foundation upon which all web pages are built.

Every HTML document follows a hierarchical structure, starting with the document type declaration, followed by the root HTML element, and then the head and body sections. This semantic structure allows browsers, search engines, and assistive technologies to understand and properly process web content.

Common HTML Tags

HTML uses various tags to structure content. Here are the most commonly used tags in web development:

Headings

HTML provides six levels of headings, from h1 (most important) to h6 (least important):

Heading Tags
<h1>Main Heading (Most Important)</h1>
<h2>Section Heading</h2>
<h3>Subsection Heading</h3>
<h4>Sub-subsection Heading</h4>
<h5>Minor Heading</h5>
<h6>Smallest Heading</h6>

Paragraphs and Text

Paragraphs and text formatting are essential for content presentation:

Text Tags
<p>This is a paragraph of text.</p>
<p>This is another paragraph with <strong>bold text</strong> and <em>italic text</em>.</p>
<p>You can also use <b>bold</b> and <i>italic</i> tags.</p>
<p>For line breaks,<br>use the br tag.</p>
<p>Use <mark>highlight</mark> to emphasize text.</p>

Lists

HTML supports both ordered (numbered) and unordered (bulleted) lists:

List Tags
<!-- Unordered List -->
<ul>
    <li>First item</li>
    <li>Second item</li>
    <li>Third item</li>
</ul>

<!-- Ordered List -->
<ol>
    <li>First step</li>
    <li>Second step</li>
    <li>Third step</li>
</ol>

<!-- Definition List -->
<dl>
    <dt>HTML</dt>
    <dd>HyperText Markup Language</dd>
    <dt>CSS</dt>
    <dd>Cascading Style Sheets</dd>
</dl>

Links

Links are fundamental to web navigation and connect different pages and resources:

Link Tags
<!-- Link to another page -->
<a href="about.html">About Us</a>

<!-- Link to external website -->
<a href="https://www.example.com" target="_blank">External Link</a>

<!-- Link to email address -->
<a href="mailto:contact@example.com">Send Email</a>

<!-- Link to phone number -->
<a href="tel:+1234567890">Call Us</a>

<!-- Link to section on same page -->
<a href="#section1">Go to Section 1</a>

Images

Images enhance web content and provide visual context:

Image Tags
<!-- Basic image -->
<img src="image.jpg" alt="Description of image">

<!-- Image with dimensions -->
<img src="logo.png" alt="Company Logo" width="200" height="100">

<!-- Responsive image -->
<img src="photo.jpg" alt="Photo description" style="max-width: 100%; height: auto;">

<!-- Image with link -->
<a href="gallery.html">
    <img src="thumbnail.jpg" alt="Gallery thumbnail">
</a>

Basic HTML Document Structure

Every HTML document follows a standard structure with essential elements:

Complete HTML Document
<!DOCTYPE html>
<html lang="en">
<head>
    <!-- Meta information about the document -->
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <meta name="description" content="A brief description of the page">
    <meta name="keywords" content="HTML, web development, tutorial">
    <meta name="author" content="Your Name">
    
    <!-- Page title (appears in browser tab) -->
    <title>My Web Page Title</title>
    
    <!-- Link to external CSS file -->
    <link rel="stylesheet" href="styles.css">
    
    <!-- Favicon -->
    <link rel="icon" href="favicon.ico" type="image/x-icon">
</head>
<body>
    <!-- Header section -->
    <header>
        <nav>
            <ul>
                <li><a href="index.html">Home</a></li>
                <li><a href="about.html">About</a></li>
                <li><a href="contact.html">Contact</a></li>
            </ul>
        </nav>
    </header>

    <!-- Main content area -->
    <main>
        <section>
            <h1>Welcome to My Website</h1>
            <p>This is the main content of the page.</p>
            
            <article>
                <h2>Article Title</h2>
                <p>Article content goes here...</p>
                <img src="article-image.jpg" alt="Article illustration">
            </article>
        </section>
        
        <aside>
            <h3>Sidebar Content</h3>
            <p>Additional information or navigation.</p>
        </aside>
    </main>

    <!-- Footer section -->
    <footer>
        <p>&copy; 2024 My Website. All rights reserved.</p>
        <p>Contact: <a href="mailto:info@example.com">info@example.com</a></p>
    </footer>

    <!-- JavaScript files -->
    <script src="script.js"></script>
</body>
</html>

Key Structure Elements Explained

  • <!DOCTYPE html>: Declares the document type as HTML5
  • <html>: Root element that contains all other elements
  • <head>: Contains metadata not visible to users
  • <meta>: Provides information about the document
  • <title>: Sets the page title shown in browser tabs
  • <body>: Contains all visible content
  • <header>: Introductory content or navigation
  • <main>: Primary content of the document
  • <section>: Thematic grouping of content
  • <article>: Self-contained piece of content
  • <aside>: Content related to main content (sidebar)
  • <footer>: Footer information

HTML Forms and Input Elements

Forms allow users to input and submit data to web servers:

    <!-- Text input -->

Semantic HTML and Best Practices

Semantic HTML uses meaningful tags that describe the content's purpose rather than just its appearance. This improves accessibility, SEO, and code maintainability:

  • Use appropriate heading hierarchy: h1 for main title, h2 for sections, etc.
  • Choose semantic elements: <article>, <section>, <nav>, <aside>
  • Always include alt attributes: For images and accessibility
  • Use labels with form inputs: Improves usability and accessibility
  • Validate your HTML: Use W3C validator to check for errors
  • Keep it clean: Proper indentation and commenting
  • Mobile-first approach: Include viewport meta tag
Semantic HTML Example
<article class="blog-post">
    <header>
        <h1>Understanding HTML Semantics</h1>
        <p>Published on <time datetime="2024-01-15">January 15, 2024</time> 
           by <address>John Doe</address></p>
    </header>
    
    <section>
        <h2>Introduction</h2>
        <p>Semantic HTML provides meaning to web content...</p>
    </section>
    
    <section>
        <h2>Benefits</h2>
        <ul>
            <li>Better accessibility</li>
            <li>Improved SEO</li>
            <li>Cleaner code</li>
        </ul>
    </section>
    
    <footer>
        <p>Tags: <span class="tag">HTML</span>, <span class="tag">Web Development</span></p>
    </footer>
</article>
